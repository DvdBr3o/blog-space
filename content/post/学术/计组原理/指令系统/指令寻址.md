| **寻址方式** | **操作数内存布局**        | **操作数**              | **访存次数** |
| -------- | ------------------ | -------------------- | -------- |
| 立即数寻址    | `Mod : Imme.Data`  | `立即数(Imme.Data)`     | 0        |
| 存储器直接寻址  | `Mod : A`          | `(A)`                | 1        |
| 存储器间接寻址  | `Mod : A`          | `((A))`              | 2        |
| 寄存器直接寻址  | `Mod : Rn`         | `(Rn)`               | 0        |
| 寄存器间接寻址  | `Mod : Rn`         | `((Rn))`             | 1        |
| PC 相对寻址  | `Mod : Rb(PC) : A` | `((PC) + A + 当前指令长)` | 1        |
| 基址寻址     | `Mod : Rb(B) : A`  | `((B) + A)`          | 1        |
| 变址寻址     | `Mod : Rb(I) : A`  | `(A + (I))`          | 1        |
| 堆栈寻址     | `SP`               |                      |          |

>[!info] 约定
>1. 上方表格内存布局中形如 `R_(NAME)` 的操作数为**寄存器**
>2. `(X)` 表示取 `X` 处存储的数据

>[!note] 基址
>基址存放在基址寄存器中, 由操作系统给出, 用户不得操作

>[!note] 各种相对寻址の地址编码
>相对寻址/基址寻址/变址寻址の**形式地址**都是**补码**
>而寄存器 PC & BR & I 的数据都是**无符号原码**
>操作时把形式地址**逻辑扩展**后直接相加，因为无符号原码的补码是他本身
